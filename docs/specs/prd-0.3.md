## v0.3 需求规格草稿（含可测试契约）

### 1. 目标（Goal）

- 产品类型：停车模拟器原型。
- 目标用户：新手司机。
- 核心目标：帮助用户判断当前停车状态与下一步动作，顺利把车停进车位。
- 版本目标：原型版，只要求能跑通基本功能且结果判定正确。

### 2. 角色（Role）

- 主角色：新手司机（单一主要使用者）。
- 使用意图：在验证自己停车思路/操作时，借助模拟器判断动作是否可行。

### 3. 触发条件（Trigger）

- 触发时机：用户想验证停车想法时可随时启动系统。
- 触发动作：进入前端界面并选择一个停车场景后开始模拟。

### 4. 输入（Input）

- 场景输入（四选一）：
1. 正常倒车入库
2. 窄路倒车入库
3. 正常侧方停车
4. 窄路侧方停车

- 每个场景采用固定标准初始位姿：
1. 初始位置固定
2. 初始朝向固定
3. 初始速度为 0

- 操作输入：
1. 方向盘控制（范围：-1 到 1）
2. 前进/后退与油门控制（范围：-1 到 1）

- 控制采样频率：20Hz。
- 交互目标：用户通过连续控制让车辆进入标定车位。

### 5. 输出（Output）

- 输出介质：仅前端界面展示，不生成任何文件。
- 核心输出：
1. 是否成功停进车位（成功/失败）
2. 失败主因标签（角度超阈值 / 越线 / 未静止）

- 结算触发机制（混合）：
1. 用户手动点击“结束”触发结算
2. 或车辆静止持续 2 秒自动结算

### 6. 边界与异常（Boundary & Exception）

- 输入中断（页面失焦/切后台）：模拟暂停，恢复后继续。
- 超时：单次模拟超过 120 秒判定失败。
- 车辆碰撞：立即判定失败。
- 车辆出界：立即判定失败。
- 成功与失败均需在前端即时展示结论。

### 7. 约束（Constraints）

- 成功判定几何阈值：
1. 车身投影 100% 在车位线内
2. 车身与车位主轴夹角 <= 8 度
3. 最终速度为 0 且持续 >= 2 秒

- 运行环境约束：
1. 桌面端 Chrome（近两个大版本）
2. 优先分辨率 1280x720 及以上

- 数据与产物约束：
1. 不生成输出文件
2. 不要求账号系统

### 8. 非功能需求（Non-functional）

- 响应性：
1. 输入到画面反馈 < 100ms
2. 结算结果展示 < 300ms

- 稳定性：
1. 单次会话不崩溃
2. 异常后可恢复到场景初始状态

- 一致性：
1. 同场景同输入序列需得到一致结果（确定性）

- 可观测性（开发调试）：
1. 允许内存级调试信息（帧率/碰撞状态/阈值命中）
2. 不落盘

### 9. 非目标（Non-goals）

- 不做高保真真实车辆动力学（如轮胎模型、悬架、ESP）。
- 不做 AI 教练/语音指导/教学文案系统。
- 不做多人、排行榜、云端存档与运营分析平台。
- 不做移动端全量适配（当前阶段聚焦桌面端）。

### 10. 决策状态

#### 10.1 已在当前版本拍板（必须现在定）

1. 成功判定阈值具体值
2. 失败条件全集（碰撞、出界、超时、手动结束）
3. 场景初始位姿固定
4. 结算触发机制（手动或静止自动）
5. 输出粒度（成功/失败 + 失败主因标签）

#### 10.2 可延后决策（后续版本再定）

1. 是否输出“下一步动作建议”
2. 是否扩展更多复杂工况（雨天、坡道、障碍随机）
3. 视觉与音效精细度目标
4. 多浏览器与移动端适配范围
5. 长期埋点与数据分析策略

### 11. 可测试契约（Testable Contract）

#### 11.1 正常路径（3 条）

1. 场景进入并成功停车（手动结算）
- Given：选择任一场景并开始模拟
- When：用户连续输入控制，车辆满足成功阈值后点击“结束”
- Then：前端在 300ms 内显示“成功停进车位”

2. 场景进入并成功停车（自动结算）
- Given：已开始模拟
- When：车辆满足成功阈值且速度为 0 持续 2 秒
- Then：系统自动结算并显示“成功停进车位”

3. 四场景可独立完成一轮
- Given：分别选择四个场景
- When：每个场景完成一次成功结算
- Then：四个场景均可从固定初始位姿启动并得到可见结算结果

#### 11.2 边界路径（3 条）

1. 角度边界
- Given：车辆在线内且速度为 0 持续 2 秒
- When：车身与车位主轴夹角恰好 8 度
- Then：结算为成功

2. 静止时长边界
- Given：车辆在线内且角度 <= 8 度
- When：速度为 0 持续时间恰好 2 秒
- Then：自动结算为成功

3. 超时边界
- Given：模拟已开始且未触发其他结算条件
- When：运行时间达到 120 秒
- Then：立即结算为失败并在前端展示

#### 11.3 失败路径（3 条）

1. 越线失败
- Given：模拟进行中
- When：车身投影任一部分越过车位线
- Then：结算为失败，主因标签为“越线”

2. 碰撞失败
- Given：模拟进行中
- When：发生车辆碰撞
- Then：立即结算为失败

3. 未静止失败（手动结束）
- Given：车辆在线内且角度满足条件，但未满足静止 2 秒
- When：用户点击“结束”
- Then：结算为失败，主因标签为“未静止”

#### 11.4 验收标准（可观察、可测试）

1. 支持四个场景入口，且每次进入均使用固定初始位置、固定朝向、初始速度 0。
2. 仅接受方向盘与前进/后退油门两类控制输入，范围均为 -1 到 1。
3. 成功判定必须同时满足：100% 在线内、角度 <= 8 度、速度为 0 持续 >= 2 秒。
4. 必须支持手动结束与静止 2 秒自动结束两种结算方式。
5. 失败条件至少覆盖：越线、碰撞、出界、超时（>=120 秒）、未静止。
6. 结算结果仅在前端界面展示，不生成任何文件。
7. 结算结果展示时延 <= 300ms；输入到画面反馈时延 < 100ms。
8. 同场景与同输入序列重复执行，结算结果一致（确定性）。
9. 页面失焦/切后台时模拟暂停，恢复后可继续并可正常结算。

#### 11.5 Definition of Done

1. 上述验收标准 9 条全部通过并有测试记录或 smoke checklist 记录。
2. 正常路径、边界路径、失败路径各 3 条全部执行通过。
3. 四个场景各至少完成 1 次成功结算和 1 次失败结算验证。
4. 无阻断级问题：崩溃、无法结算、结果不显示、判定与阈值不一致。
5. 不生成文件输出，结算结果仅在前端可见。

#### 11.6 仍存在的需求风险

1. 目标描述包含“下一步动作判断”，但当前契约仅覆盖成败判定与失败主因标签。
2. 失败主因标签当前定义为 3 类，碰撞/出界/超时是否也需要标签仍可能产生口径差异。
3. 性能指标缺少统一测试机与负载条件，不同设备下可比性有限。
4. 碰撞与出界的几何边界尚未进一步量化，临界帧可能出现争议。
5. 当前仅约束桌面 Chrome，跨浏览器行为一致性风险留在后续版本。

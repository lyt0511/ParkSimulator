## v0.1 需求规格草稿

### 1. 目标（Goal）

- 产品类型：停车模拟器原型。
- 目标用户：新手司机。
- 核心目标：帮助用户判断当前停车状态与下一步动作，顺利把车停进车位。
- 版本目标：原型版，只要求能跑通基本功能且结果判定正确。

### 2. 角色（Role）

- 主角色：新手司机（单一主要使用者）。
- 使用意图：在验证自己停车思路/操作时，借助模拟器判断动作是否可行。

### 3. 触发条件（Trigger）

- 触发时机：用户想验证停车想法时可随时启动系统。
- 触发动作：进入前端界面并选择一个停车场景后开始模拟。

### 4. 输入（Input）

- 场景输入（四选一）：
1. 正常倒车入库
2. 窄路倒车入库
3. 正常侧方停车
4. 窄路侧方停车


- 操作输入：
1. 方向盘控制
2. 前进/后退与油门控制

- 交互目标：用户通过连续控制让车辆进入标定车位。

### 5. 输出（Output）

- 输出介质：仅前端界面展示，不生成任何文件。
- 核心输出：
1. 是否成功停进车位（成功/失败）
2. 失败主因标签（角度超阈值 / 越线 / 未静止）

- 结算触发机制（混合）：
1. 用户手动点击“结束”触发结算
2. 或车辆静止持续 2 秒自动结算

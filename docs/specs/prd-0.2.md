## v0.2 需求规格草稿（已按默认值拍板）

### 1. 目标（Goal）

- 产品类型：停车模拟器原型。
- 目标用户：新手司机。
- 核心目标：帮助用户判断当前停车状态与下一步动作，顺利把车停进车位。
- 版本目标：原型版，只要求能跑通基本功能且结果判定正确。

### 2. 角色（Role）

- 主角色：新手司机（单一主要使用者）。
- 使用意图：在验证自己停车思路/操作时，借助模拟器判断动作是否可行。

### 3. 触发条件（Trigger）

- 触发时机：用户想验证停车想法时可随时启动系统。
- 触发动作：进入前端界面并选择一个停车场景后开始模拟。

### 4. 输入（Input）

- 场景输入（四选一）：
1. 正常倒车入库
2. 窄路倒车入库
3. 正常侧方停车
4. 窄路侧方停车

- 每个场景采用固定标准初始位姿：
1. 初始位置固定
2. 初始朝向固定
3. 初始速度为 0

- 操作输入：
1. 方向盘控制（范围：-1 到 1）
2. 前进/后退与油门控制（范围：-1 到 1）

- 控制采样频率：20Hz。
- 交互目标：用户通过连续控制让车辆进入标定车位。

### 5. 输出（Output）

- 输出介质：仅前端界面展示，不生成任何文件。
- 核心输出：
1. 是否成功停进车位（成功/失败）
2. 失败主因标签（角度超阈值 / 越线 / 未静止）

- 结算触发机制（混合）：
1. 用户手动点击“结束”触发结算
2. 或车辆静止持续 2 秒自动结算

### 6. 边界与异常（Boundary & Exception）

- 输入中断（页面失焦/切后台）：模拟暂停，恢复后继续。
- 超时：单次模拟超过 120 秒判定失败。
- 车辆碰撞：立即判定失败。
- 车辆出界：立即判定失败。
- 成功与失败均需在前端即时展示结论。

### 7. 约束（Constraints）

- 成功判定几何阈值：
1. 车身投影 100% 在车位线内
2. 车身与车位主轴夹角 <= 8 度
3. 最终速度为 0 且持续 >= 2 秒

- 运行环境约束：
1. 桌面端 Chrome（近两个大版本）
2. 优先分辨率 1280x720 及以上

- 数据与产物约束：
1. 不生成输出文件
2. 不要求账号系统

### 8. 非功能需求（Non-functional）

- 响应性：
1. 输入到画面反馈 < 100ms
2. 结算结果展示 < 300ms

- 稳定性：
1. 单次会话不崩溃
2. 异常后可恢复到场景初始状态

- 一致性：
1. 同场景同输入序列需得到一致结果（确定性）

- 可观测性（开发调试）：
1. 允许内存级调试信息（帧率/碰撞状态/阈值命中）
2. 不落盘

### 9. 非目标（Non-goals）

- 不做高保真真实车辆动力学（如轮胎模型、悬架、ESP）。
- 不做 AI 教练/语音指导/教学文案系统。
- 不做多人、排行榜、云端存档与运营分析平台。
- 不做移动端全量适配（当前阶段聚焦桌面端）。

### 10. 决策状态

#### 10.1 已在当前版本拍板（必须现在定）

1. 成功判定阈值具体值
2. 失败条件全集（碰撞、出界、超时、手动结束）
3. 场景初始位姿固定
4. 结算触发机制（手动或静止自动）
5. 输出粒度（成功/失败 + 失败主因标签）

#### 10.2 可延后决策（后续版本再定）

1. 是否输出“下一步动作建议”
2. 是否扩展更多复杂工况（雨天、坡道、障碍随机）
3. 视觉与音效精细度目标
4. 多浏览器与移动端适配范围
5. 长期埋点与数据分析策略

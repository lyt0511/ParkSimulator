# PRD-0.3 实施计划（已固定）

## 总目标

- 基于 `docs/specs/prd-0.3.md`，以“可测试、可回滚、可小步合并”为原则完成原型版最小可交付。
- 本轮聚焦：四场景、输入语义、成功判定、结算机制、6类失败原因前端可见输出。
- 交付底线：每个切片均可独立通过 `npm test` 与 `npm run build`，且可独立回滚。

## 范围锁定

- 需求源路径固定为：`docs/specs/prd-0.3.md`
- 本轮不补 `lint`。
- 失败原因固定为 6 类可见输出：
  - `angle_over_limit`
  - `cross_line`
  - `not_still`
  - `collision`
  - `out_of_bounds`
  - `timeout`
- 非功能验证使用当前默认方式（当前环境 + 现有 npm 脚本）统一执行。

## 切片1：顶层骨架

- 目标：
  - 建立最小工程骨架，覆盖菜单入口、游戏状态壳、场景 schema 壳、输入常量、测试脚本骨架。
- 主要改动文件：
  - `src/ui/*`
  - `src/core/*`
  - `src/scenes/*`
  - `tests/contract/*`
  - `package.json`（仅脚本入口，不新增 lint）
- 切片测试：
  - 合同测试骨架可执行（至少能跑通结构性检查）。
  - 验证目录与关键文件落点正确。
  - 执行 `npm test`、`npm run build` 并记录结果。
- 风险：
  - 骨架阶段过度抽象，提前引入不必要层次。
- 回滚点：
  - 以本切片提交为单位整体回滚。

## 切片2：场景 schema 与固定初始位姿

- 目标：
  - 锁定 4 个场景及固定初始状态，为后续运动与判定提供稳定输入基线。
- 主要改动文件：
  - `src/scenes/index.ts`
  - `tests/contract/*`（场景与初始状态契约检查）
- 切片测试：
  - 校验 4 个场景均存在。
  - 校验每个场景初始状态字段完整，且初始速度为 `0`。
  - 执行 `npm test`、`npm run build`。
- 风险：
  - schema 字段频繁变更导致下游切片重复改动。
- 回滚点：
  - 仅回滚 `src/scenes/*` 与对应测试改动。

## 切片3：车辆运动与方向盘语义

- 目标：
  - 定义方向盘与前进/后退油门输入语义，以及状态更新语义。
- 主要改动文件：
  - `src/core/simulator.ts`
  - `src/core/rules.ts`（仅与运动语义相关常量）
  - `tests/contract/*`（输入范围与语义契约）
- 切片测试：
  - 校验输入范围 `[-1, 1]`。
  - 校验采样频率与语义定义一致。
  - 校验同输入序列输出一致（确定性基础检查）。
  - 执行 `npm test`、`npm run build`。
- 风险：
  - core 与 UI 对输入语义解释不一致。
- 回滚点：
  - 独立回滚运动语义相关 `src/core/*` 提交。

## 切片4：成功判定与结算机制

- 目标：
  - 实现成功三条件与混合结算机制（手动结束 / 静止自动结束）。
- 主要改动文件：
  - `src/core/rules.ts`
  - `src/core/simulator.ts`
  - `tests/contract/*`（正常路径与边界路径）
- 切片测试：
  - 角度 `8` 度边界判定。
  - 静止 `2` 秒边界判定。
  - 手动结束与自动结束均可触发并输出结果。
  - 执行 `npm test`、`npm run build`。
- 风险：
  - 阈值边界附近出现抖动或判定歧义。
- 回滚点：
  - 独立回滚结算相关提交，不影响场景切片。

## 切片5：6 类失败原因可见输出

- 目标：
  - 打通 6 类失败原因到前端可见输出。
- 主要改动文件：
  - `src/core/rules.ts`
  - `src/core/simulator.ts`
  - `src/ui/app.ts`
  - `tests/contract/*`（失败路径契约）
- 切片测试：
  - 逐条验证 6 类失败原因可触发、可见、标签一致。
  - 校验核心枚举与 UI 展示文案/标签映射一致。
  - 执行 `npm test`、`npm run build`。
- 风险：
  - 枚举值与展示值不一致造成契约漂移。
- 回滚点：
  - 先回滚 UI 映射，再按需回滚 core 映射。

## 切片6：测试加固、调试钩子与交付整理

- 目标：
  - 补齐最终测试覆盖、最小调试钩子与交付清单，不扩展功能范围。
- 主要改动文件：
  - `tests/contract/*`
  - `src/ui/app.ts` 或 `src/core/*`（仅最小调试钩子）
  - `docs/specs/prd-0.3.md`（如需补测试映射附录）
- 切片测试：
  - 汇总执行正常/边界/失败路径 smoke checklist。
  - 校验 DoD 条目可勾选、可复现。
  - 执行 `npm test`、`npm run build`。
- 风险：
  - 调试钩子变成功能扩展，偏离本轮范围。
- 回滚点：
  - 可单独回滚“调试与整理”提交，不影响主功能。

## 过程护栏

- 不允许一次改很多层。
- 不允许把“顺手优化”混入当前功能。
- 每个切片优先保证测试可执行与回滚可行。
